Projects Overview

A project in Code on the Go contains everything that defines your workspace for an app, including source code and resources to help test and build your app.

When you start a new project, Code on the Go creates the necessary structure for all your files and makes them visible in the Project window. To open the Project window, swipe right on the screen or tap the upper-left menu.

The content on this page is for demo purposes only. It may not make sense to have all the module and library stuff default on the page--there's a good chance that makes more sense when you're actually IN the IDE, not picking a template.

Create a project

To create a project, follow these steps on the Get started page:

  1. Tap Create project. The New project page is displayed.
  2. Select a template. Learn more about templates here. A configuration page for the project is displayed.
  3. In the App name field, enter a name for your app. The Package name field will be updated with the name of the app.
  4. The project will be saved in the default directory. To change the directory, tap Save location and select a different directory.
  5. Tap Create project. Code on the Go will create new project files and directories. Please be patient while Code on the Go sets up the project. When the process is finished, Project Initialized will be displayed at the bottom of the screen.
  6. To see the file structure for your app and begin writing code, swipe from the left side of the screen to the right. The file tree for your project will be displayed.

Delete a project

WARNING: Deleting a project permanently removes all of the files and directories associated with your project. A deleted project can't be restored.

To delete a project, follow these steps on the Get startedpage:

  1. Tap Delete a project. SOMETHING WILL BE DISPLAYED.
  2. DO WHATEVER YOU DO TO LOCATE AND SELECT A PROJECT TO DELETE.
  3. CLICK WHAT YOU CLICK. A CONFIRMATION DIALOG SHOULD DEFINITELY BE DISPLAYED.
  4. CLICK WHAT YOU CLICK TO THE CONFIRM. The project will be deleted and the Get started

    page will be displayed.

Modules Overview

A module is a collection of source files and build settings that let you divide your project into discrete units of functionality. Your project can have one or many modules, and one module can use another module as a dependency. You can independently build, test, and debug each module.

Additional modules are useful when creating code libraries within your own project or when you want to create different sets of code and resources for different device types, such as phones and wearables, but keep all the files scoped within the same project and share some code. Some common modules are described below.

Module Overview
Library modules Library modules provide a container for your reusable code, which you can use as a dependency in other app modules or import into other projects. Structurally, a library module is the same as an app module, but when built, it creates a code archive file instead of an APK, so it can't be installed on a device.
Android app modules These modules provide containers for your app's source code, resource files, and app-level settings, such as the module-level build file and Android Manifest file. When you create a new project, the default app module is named "app." This may be a link to a help page that's entirely about modules.

Project files

By default, Code on the Go displays your project files in the Android view. This view is organized by modules and file types to simplify navigation between key source files of your project and hides some files or directories that are not commonly used.

Files are organized in the following way:

File type Meaning
manifests Contains the AndroidManifest.xml file. Learn more about manifests.
java Contains the Kotlin and Java source code files, separated by package names, including JUnit test code.
res Contains all non-code resources, such as UI strings and bitmap images, divided into corresponding subdirectories. For more information about possible resource types, see App resources overview, a page that doesn't exist yet...but it will.
